Schedule

Declarative schedules live in config.json's schedules[] block. Save here, then run dbx schedule sync from the terminal to install, update, or orphan the launchd / systemd units.

Declarative (config.json)

when accepts hourly, daily, daily@H, weekly, weekly@D:H, or a raw cron expression. Next run is computed from the schedule; last run reflects the newest backup for that host / database. Retention is applied at backup / clean time and isn't yet configurable per schedule. Next/last run reflect the last saved state — save to refresh after editing.

unsaved changes invalid rows (see fields)

Installed units

launchd plists on macOS, systemd timers on Linux. Units installed before the DbxScheduleExpression marker show ? for when.

Sync plan

What dbx schedule sync would do given the saved declarative + installed state. Reflects the last saved declarative, not the unsaved edits above. Run dbx schedule sync --apply from the CLI to act on it.
